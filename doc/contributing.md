# Guide de Contribution

## Conventions de Commit
- Format : `[SPRINT-ID] Description courte du changement`.
- Exemple : `[SPRINT-001] Ajout de la fonction de lecture JSON`.

## Style de Code
- Respecter les conventions ESLint.
- Prioriser les bonnes pratiques de sécurité, surtout pour les données sensibles.

## Tests 
- `backend/tests`
- Ajouter des tests pour chaque nouvelle fonctionnalité ou correction.
- Utiliser `jest` pour les tests unitaires et `cypress` pour les tests fonctionnels.
